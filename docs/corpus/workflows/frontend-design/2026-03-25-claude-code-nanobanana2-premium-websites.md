## Key Takeaways

1. **Nano Banana 2 enables scroll-stopping 3D animated hero sections** by generating a start image and an exploded/deconstructed version, then using Kling 3.0 to create a video transition between them. This video gets embedded as a scroll-driven animation on the website.

2. **Brand extraction via Firecrawl gives you instant design DNA** -- scrape a client's existing site to pull colors, typography, logos, and tone. Feed this into Claude Code so the generated site matches their identity from the start.

3. **HTML extraction + Claude Code = instant structural cloning** -- download any website's HTML, feed it to Claude Code, and recreate the same scaffold with new content and animated assets layered in. One-shot rebuild of a client-quality site.

4. **The full pipeline is five steps**: (1) brand identity extraction, (2) image generation with Nano Banana 2 using reference images, (3) video transition via Kling 3.0, (4) website build in Claude Code / Anti-Gravity with scroll-driven animation, (5) deploy via Vercel + GitHub.

5. **Claude Code skills are the automation layer** -- custom skills for asset generation prompting, 3D website building, and SEO auditing. These can be shared, installed, and reused across projects.

6. **SEO optimization is a separate skill-driven pass** -- a Claude Code SEO skill crawls the deployed site, scores it, generates a full audit report (page-by-page breakdown, internal linking, keyword analysis, action plan), and outputs an HTML report.

7. **The MONEY framework for selling websites**: Map (find boring niches), Obtain (scrape/enrich leads), Nail (build the site), Execute (give it away free), Yield (sell recurring services on top).

## Detailed Notes

### Brand Identity Extraction (Step 1)
- Use Firecrawl.dev: navigate to Scrape > Markdown > Branding
- Enter client URL, it extracts colors, logos, typography, brand assets
- Firecrawl has an API -- can be integrated into Anti-Gravity and Claude Code skills for doing this at scale
- This is the "beating heart" of the website -- everything downstream references this

### Image Generation with Nano Banana 2 (Step 2)
- Go to Nano Banana 2, select 16x9 aspect ratio
- Need two images: an assembled/complete version and a deconstructed/exploded version
- Use Claude Code's asset generation skill to create optimized prompts for the image generation
- Workflow: `/skill-creator` to build the 3D asset generator, then `/scroll-stop-prompter` to generate prompts
- Include client logo and reference images (e.g., their actual van, product photos) as inputs
- Generate at minimum 2K resolution (1K looks amateur)
- Run 4 iterations minimum to get good options
- Specify clean white background so edges stay clean for web embedding
- Use the "reference" feature: generate the assembled image first, reference it when generating the exploded version so they share visual continuity

### Video Transition via Kling 3.0 (Step 3)
- Use Kling 3.0 (accessed via Hixelon or directly via API)
- Upload start frame (assembled image) and end frame (exploded image)
- Paste the video transition prompt generated by Claude Code
- Settings: no multi-shot, no enhance, 7 seconds duration
- Result: animated transition showing items flying out, furniture moving, etc.
- Can also be done entirely from Claude Code with an API key (no GUI needed)
- Iterate 4-5 versions to get smooth transitions

### Website Build with Claude Code + Anti-Gravity (Step 4)
- Install two Claude Code skills: "Asset Generation" and "3D Website Builder"
- Create new project folder in Anti-Gravity (VS Code)
- Activate Claude Code via the extension sidebar
- Use `/scroll-stop-builder` skill to walk through the build
- Tell Claude where the downloaded video file is -- it handles integration
- Tip: `Shift` toggles between "ask before edits," "edit automatically," and "plan mode" -- use "edit automatically" for speed
- Result: full website with brand colors, scroll-driven 3D animation, mobile responsive layout

### HTML Extraction and Structural Cloning
- Use an HTML website extractor tool (Google "HTML website extractor")
- Enter existing client URL, download as HTML
- Feed the HTML to Claude Code: "recreate this with new copy and scroll-stopping animation"
- Key: grab their logo and typography so it looks/feels like the original
- Demonstrated with anyvan.co.uk and rocket.net
- The rocket.net example used Nano Banana 2 to replace static images with animated versions while keeping the same page structure -- same layout, different visual treatment
- One-shot rebuild is possible for structurally sound sites

### Deployment via Vercel + GitHub (Step 5)
- Connect Anti-Gravity to both GitHub and Vercel via MCP config
- Vercel token: Settings > Tokens > create new token
- Tell Anti-Gravity: "add the Vercel MCP with this API key" and reference @mcp_config
- Prompt: "create a GitHub repo, publish it, create a Vercel deployment"
- Result: live site with custom domain capability
- Domain assignment: Vercel dashboard > Domains > assign
- Analytics: enable on Vercel, paste tracking code, tell Anti-Gravity to add it
- Speed insights also available through Vercel

### SEO Optimization (Separate Pass)
- Import SEO optimization skill into Claude Code
- Run `/seo-strategy` to kick off full audit
- Process: fetch homepage, discover internal pages, fetch robots.txt/sitemap, analyze each page, cross-page analysis, score categories, generate HTML report
- Can specify target keywords
- Output includes: executive summary, page-by-page breakdown, sitewide issues, technical details, content strategy, internal linking map, keyword analysis, full action plan with implementation checklist
- Their demo scored a "D" on first pass -- shows the audit catches real issues
- Multi-page SEO: prompt Claude to create pages with full meta descriptions, OG tags, canonical URLs, Twitter cards, JSON-LD structured data, responsive design, cross-linked navigation, shared CSS design system

### Multi-Page Site Generation
- Prompt: "Look at my current site, understand the design language, ask me which pages to create, match existing design exactly, add full SEO, include structured data, make responsive, update navigation across all pages, run full SEO audit"
- Demo created: About page, Testimonials page, Team page
- All pages share same design system, navigation links between them
- Anti-Gravity handles the page creation while Claude Code handles SEO auditing in parallel

### The MONEY Framework (Sales Methodology)
- **M - Map**: Find boring niches (moving companies, plumbers, etc.)
- **O - Obtain**: Script lead generation, enrich with names/contact info
- **N - Nail**: Build the beautiful websites using this pipeline
- **E - Execute**: Give websites away for free (can automate at scale)
- **Y - Yield**: Sell recurring revenue services (SEO, analytics, maintenance)
- Sells a masterclass walking through this step by step

## Applicable to Us

**For the Polylogic AI consulting landing page, several techniques are directly applicable:**

1. **Scroll-driven animation as hero section**: The Nano Banana 2 + Kling 3.0 pipeline for creating animated hero sections is worth testing. A scroll-stopping 3D animation of an AI agent workspace or Claude Code terminal coming to life could differentiate the Polylogic site from typical consulting pages.

2. **Structural cloning as starting point**: The HTML extraction technique is practical -- find 2-3 top-tier consulting agency sites (McKinsey Digital, Bain, or AI-native agencies), extract their HTML, and use it as scaffolding for Polylogic's layout. This saves design time while ensuring professional structure.

3. **Vercel + GitHub deployment pipeline**: The MCP-based deployment flow (Anti-Gravity > GitHub > Vercel) is the exact stack we should use. Free tier Vercel is generous, and the analytics/speed insights come built in.

4. **SEO audit skill as quality gate**: Before launching the Polylogic site, run the SEO audit skill to catch issues. The structured data, OG tags, and JSON-LD generation are table stakes for a consulting site that needs to rank.

5. **Multi-page generation with shared design system**: The prompt pattern for generating multiple pages while maintaining design consistency is useful -- Polylogic will need at minimum: landing page, services, case studies, about, contact.

6. **What to skip**: The "give websites away free, sell recurring services" model is not our play. Polylogic is a consulting agency, not a web dev shop. But the production techniques (brand extraction, animated assets, one-shot site builds) are directly transferable to building our own site quickly and professionally.
